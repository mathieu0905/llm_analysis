Abstract

Code analysis is a fundamental problem in Software Engineering (SE), playing a crucial role in tasks such as debugging, performance optimization, and security assessment. Human developers typically approach code analysis through syntax understanding, static behavior analysis, and dynamic behavior reasoning. Traditional code analysis tools, while effective, are often limited by high false positive/negative rates, language specificity, and complex configurations. Recently, large language models (LLMs) have emerged as promising tools for automating various code-related tasks. However, their capabilities for addressing code analysis tasks remain underexplored.

This paper investigates the potential of nine state-of-the-art LLMs for code analysis within their internal knowledge. We structure our study around three aspects of code analysis aligned with human practices: syntax understanding, static behavior analysis, and dynamic behavior reasoning. Using datasets spanning multiple programming languages (C, Java, Python, Solidity), we evaluate the performance of LLMs on nine representative tasks, such as Abstract Syntax Tree (AST) generation, Control Flow Graph (CFG) construction, and data dependency analysis. Our results reveal that while LLMs excel in syntax parsing and show initial promise in static analysis, their dynamic reasoning capabilities remain limited. These findings offer insights into how LLMs can serve as general-purpose code analyzers and clarify directions for future enhancement of LLM-based code analysis.
